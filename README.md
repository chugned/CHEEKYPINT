# CheekyPint 🍺

**Your social pub diary.** Log your pub visits, remember your favourite haunts, and keep
friendly standings with your mates — one tap, and cheers.

CheekyPint is a playful-but-responsible social beer diary for adults. It is a *diary and pub
memory* app, **not** a drinking game: there are no streaks, no "drink more" nudges, no global
rankings, and no rewards for heavy consumption. See [Responsible drinking](docs/RESPONSIBLE_DRINKING.md).

> Status: MVP scaffold. The domain core and backend are implemented and **verified with
> automated tests**; the SwiftUI app is complete in structure and requires Xcode to build/run.

---

## What's in here

```
cheekypint/
  CheekyPintCore/     Foundation-only domain logic (counting, privacy, leaderboard, QR, safety)
                      → 100% unit-tested, runs on macOS without Xcode
  CheekyPint/         The SwiftUI iOS app (App, Core services, Features, Resources)
  CheekyPint.xcodeproj / project.yml   Xcode project (generated by XcodeGen)
  CheekyPintTests / CheekyPintUITests  App-level XCTest targets
  supabase/           PostgreSQL migrations, RLS, RPC functions, seed, Edge Functions, tests
  docs/               Architecture, database, security, privacy, App Store, legal templates
```

Architecture in one line: **all counting/privacy/validation rules live in `CheekyPintCore`
(tested), the backend enforces them again via RLS + `SECURITY DEFINER` RPCs (tested), and the
SwiftUI app is a thin, well-structured client over both.** See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Quick start

### 1. Domain core (no Xcode needed)

```bash
cd CheekyPintCore
swift build
swift run corecheck      # dependency-free verification of the domain logic (56 checks)
# In Xcode / a swift.org toolchain you can also run the full XCTest suite:
#   swift test
```

### 2. Backend (Supabase)

Local, with the Supabase CLI:

```bash
brew install supabase/tap/supabase
supabase start
supabase db reset        # applies migrations + seed.sql
supabase functions serve # runs the delete-account Edge Function
```

Or verify the schema on any local Postgres **without** Supabase/Docker:

```bash
brew install postgresql@16
cd supabase/tests && ./run_local_pg.sh   # applies everything + runs RLS/RPC allow-deny tests
```

Seed users (password `password123`): `alice@`, `barnaby@`, `ceri@`, `dev@cheekypint.test`.

### 3. iOS app (needs Xcode 16+)

```bash
brew install xcodegen         # only if you change the project structure
xcodegen generate             # (re)generate CheekyPint.xcodeproj — already committed
open CheekyPint.xcodeproj
```

Then set your Supabase URL + anon key in the environment configs and pick a scheme:

- `CheekyPint/Resources/Config/Development.xcconfig` → local Supabase (`http://127.0.0.1:54321`)
- `…/Staging.xcconfig`, `…/Production.xcconfig` → your hosted projects

Select the **CheekyPint** scheme and a **Debug** (Development) build to run against local Supabase.
See [docs/APP_STORE_SUBMISSION.md](docs/APP_STORE_SUBMISSION.md) for Sign in with Apple, associated
domains, and signing.

## Configuration

| Setting | Where | Notes |
|---------|-------|-------|
| Supabase URL / anon key | `*.xcconfig` → Info.plist | Non-secret; anon key is safe to commit. |
| Service-role key | Edge Function env only | **Never** in the app. |
| Bundle IDs | `*.xcconfig` | `app.cheekypint.CheekyPint[.dev/.staging]` |
| URL scheme / universal links | Info.plist + entitlements | `cheekypint://`, `applinks:cheekypint.app` |

Copy `.env.example` to `.env` for backend/tooling secrets. Never commit real secrets.
